Richtungs-Sprites

Das letzte und kompakteste Beispiel im Kurs: eine Figur, die pro Tastendruck ihre Position und ihre Blickrichtung ändert — ganz ohne Game-Loop. Original: movement.fla + ninja_front/back/left/right.swf.

Kein ENTER_FRAME nötig — Bewegung direkt im Event

Anders als fast jede andere Übung in diesem Kurs kommt movement.fla komplett ohne Game-Loop aus: Bewegung und Richtungswechsel passieren direkt innerhalb des KeyboardEvent.KEY_DOWN-Handlers. Jeder Tastendruck verschiebt die Figur um einen festen Betrag und wechselt sofort das passende Sprite.

ActionScript 3 
function keyDown(_event) {
  switch(_event.keyCode) {
    case Keyboard.LEFT:
    case Keyboard.A:
      character_mc.x -= 5;
      character_mc.gotoAndStop("left");
      break;
    // ... right, up, down
  }
}
JavaScript 
function keyDown(e) {
  const dir = map[e.key];
  if (dir === "left") character.x -= STEP;
  // ... right, back, front
  character.direction = dir;
  render();
}

Das hat einen sichtbaren Effekt: Die Bewegung ist an die Wiederholrate der Tastatur gekoppelt (also an das Betriebssystem), nicht an die Bildwiederholrate. Ein guter Kontrast zu praktisch jeder anderen Übung in diesem Kurs, die zeitbasiert pro Frame bewegt — manchmal ist "so einfach wie möglich" genau richtig, besonders für ein Demo dieser Größe.

Vier Sprites statt einer Rotation

Statt die Figur zu drehen (wie es rotation könnte), tauscht das Original komplett das Sprite aus — gotoAndStop("left"/"right"/"up"/"down") zeigt vier völlig unterschiedlich gezeichnete Frames. Das ist der klassische Ansatz für Top-Down-Spiele mit gerichteten Charakteren (Zelda-Stil): vier (oder acht) fest gezeichnete Blickrichtungen statt einer frei rotierenden Figur.

Eigene Grafiken statt Original-Sprites

Die vier ninja_*.swf-Dateien waren winzige, im Animate-Editor gezeichnete Vektorformen ohne extrahierbare Bilddaten (ähnlich wie schon bei den Dreiecks-Grafiken im Buch-Projekt) — hier ersetzt durch vier selbst gezeichnete Blickrichtungen (Augen sichtbar von vorne, Profil-Andeutung seitlich, Haarwirbel von hinten).

WASD oder Pfeiltasten drücken — jeder Tastendruck bewegt die Figur um 5 Pixel und wechselt sofort die Blickrichtung.
