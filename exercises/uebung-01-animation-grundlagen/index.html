Animation-Grundlagen

Drei Grundtechniken der Animation in Adobe Animate — und ihre direkten Web-Entsprechungen: Frame-by-Frame, klassisches Tweening, und skriptgesteuerte Zustände. Original: strichmännchen.fla, uebung1.fla, testAnimSkripting.fla.

Drei Techniken, drei Web-Entsprechungen

Animate bot historisch drei Wege, etwas zu animieren — und alle drei haben ein Gegenstück im Web:

  • Frame-by-Frame — jedes Einzelbild von Hand gezeichnet, keine Berechnung dazwischen. Web-Entsprechung: eine Liste fertiger Zustände/Sprites, per Timer durchgeschaltet.
  • Klassisches Tweening — nur Start- und Endzustand festlegen, das Programm berechnet den Rest. Web-Entsprechung: eine Interpolationsformel (linear oder mit Easing) über die Zeit.
  • Skriptgesteuerte Animation — Code entscheidet, welcher benannte Frame/Zustand gerade laufen soll. Web-Entsprechung: eine Zustandsvariable plus requestAnimationFrame.

Frame-by-Frame: keine Zwischenschritte

Das Strichmännchen im Original besteht aus vier von Hand gezeichneten Posen, die im Rhythmus der Bildrate durchgeschaltet werden — genau das macht auch demoFrameByFrame: ein Array von vier Posen, ein setInterval(), das zur nächsten weiterschaltet. Kein Zwischenwert wird berechnet.

Tweening: eine Formel für alle Zwischenschritte

Animate interpoliert zwischen zwei Keyframes automatisch — im Web übernimmt das eine kleine Easing-Funktion. Position, Rotation und Skalierung werden alle mit demselben Fortschrittswert t berechnet.

Animate (visuell, Keyframes) 
Keyframe 1: x=90,  rot=0°,  scale=0.6
Keyframe 2: x=470, rot=360°, scale=1.4
→ Zwischenschritte automatisch
   von Animate berechnet
JavaScript (Formel) 
const p = easeInOutQuad(t);
const x = 90 + p * (470-90);
const rotation = p * 360;
const scale = 0.6 + p * 0.8;

Skriptgesteuert: das doAction()-Muster

testAnimSkripting.fla führt ein Muster ein, das den gesamten restlichen Kurs durchzieht: eine Funktion doAction(label), die zu einem benannten Zustand wechselt. Später (Übung 3 und danach) wird genau dieses Muster für Lauf-/Steh-/Sprung-Animationen einer Spielfigur wiederverwendet.

ActionScript 3 
function doAction(_action: String) {
  var nextLabel = "Idle";
  switch(_action) {
    case "translate":
      nextLabel = "Translation";
      break;
    // ...
  }
  gotoAndPlay(nextLabel);
}
JavaScript 
function doAction(newAction) {
  action = newAction;
  t0 = performance.now();
}
// im Game-Loop:
if (action === "translate") { ... }
else if (action === "rotate") { ... }
Zwischen den drei Demos oben wechseln. Bei Demo 3 stehen rechts drei Knöpfe zur Verfügung, um doAction() live auszuprobieren.
