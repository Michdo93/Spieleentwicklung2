Event-Listener & Zustände

Erstmals steuert Code die Figur aus Übung 2 — zunächst nur per Zufall, aber das ist der erste Schritt weg von reiner Handanimation. Original: eventListener.fla, uebung3.fla.

ENTER_FRAME: der Herzschlag jeder Animate-Anwendung

Event.ENTER_FRAME ist das AS3-Äquivalent zu requestAnimationFrame — eine Funktion, die bei jedem gezeichneten Bild erneut aufgerufen wird. Demo 1 zeigt das in Reinform: ein Quadrat, dessen Position, Rotation und Skalierung sich jeden Frame ein kleines Stück ändern.

ActionScript 3 
this.addEventListener(
  Event.ENTER_FRAME, update);

function update(_event: Event) {
  square_mc.x += 5;
  square_mc.rotation += 30;

  if(square_mc.rotation == 180) {
    square_mc.scaleX -=
      square_mc.scaleX/10;
    square_mc.scaleY -=
      square_mc.scaleY/10;
  }

  if(square_mc.x > stage.stageWidth
      + square_mc.width/2) {
    square_mc.x =
      -square_mc.width/2;
  }
}
JavaScript 
function frame() {
  x += 2.2;
  rotation += 13;

  if (rotation >= 180
      && rotation < 193) {
    scale -= scale/10;
  }

  if (x > W + 30) x = -30;

  requestAnimationFrame(frame);
}

Ein Detail lohnt sich hervorzuheben: Das Original prüft rotation == 180 — das trifft nur exakt einmal, weil die Rotation in 30°-Schritten springt (0, 30, 60, … 180, 210, …). Bei unserer Framerate mit krummeren Schritten würde ein exakter Vergleich fast nie zutreffen — deshalb wird hier stattdessen ein kleines Intervall (rotation >= 180 && rotation < 193) geprüft. Ein kleines, aber lehrreiches Detail beim Übergang von diskreten zu kontinuierlichen Frame-Schritten.

Die Figur bekommt ihre erste Steuerung — per Zufall

uebung3.fla lässt unsere Charakterfigur (Übung 2) erstmals von Code statt von Buttons steuern: alle paar Sekunden entscheidet Math.random(), ob sie läuft (70% Wahrscheinlichkeit), steht (20%) oder springt (10%). Das ist noch keine Spielsteuerung — aber es ist der erste Schritt, bei dem eine Zeile Code statt eines Klicks eine Animation auswählt.

ActionScript 3 
var random = Math.random();

if(random <= 0.7) {
  charakter_mc.doAction("walk");
}
if(random > 0.7 && random <= 0.9) {
  charakter_mc.doAction("idle");
}
if(random > 0.9 && random <= 1) {
  charakter_mc.doAction("jump");
}
JavaScript 
const random = Math.random();
let action;
if (random <= 0.7) action = "walk";
else if (random <= 0.9) action = "idle";
else action = "jump";
rig.doAction(action);

doAction() selbst wurde ebenfalls unverändert übernommen: es wechselt den Zustand nur, wenn er sich tatsächlich ändert (if (this.currentAnimation !== action)) — sonst würde ein wiederholtes "walk" die Laufanimation ständig neu starten.

Bei Demo 2 rechts unten das Ausgabe-Panel beobachten — dort steht, welcher Zufallswert zu welcher Aktion geführt hat.
