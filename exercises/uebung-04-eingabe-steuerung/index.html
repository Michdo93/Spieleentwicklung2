Eingabe & Charakter-Steuerung I

Maus- und Tastatur-Grundlagen, dann fünf unabhängige Charaktere, die sich jetzt wirklich über die Bühne bewegen — und deren Animationen ihr eigenes Ende selbst zurückmelden. Original: vl05.fla, uebung4.fla.

Maus- und Tastatur-Events: die Grundbausteine

vl05.fla zeigt beide Eingabearten nebeneinander: MouseEvent.CLICK auf ein Objekt, und KeyboardEvent.KEY_DOWN mit einem switch über Keyboard.LEFT/RIGHT/UP/DOWN (plus WASD als Alternative). Beides übersetzt sich nahezu wörtlich:

ActionScript 3 
rect_mc.addEventListener(
  MouseEvent.CLICK, clicked);

stage.addEventListener(
  KeyboardEvent.KEY_DOWN, keyDown);

function keyDown(_event) {
  switch(_event.keyCode) {
    case Keyboard.UP:
    case Keyboard.W:
      trace("^");
      break;
    // ...
  }
}
JavaScript 
canvas.addEventListener(
  'click', clicked);

window.addEventListener(
  'keydown', onKey);

function onKey(e) {
  const map = {
    ArrowUp: '^', w: '^', /* ... */
  };
  if (map[e.key]) trace(map[e.key]);
}

Vom Zufallstimer zur selbstmeldenden Animation

In Übung 3 entschied ein festes Zeitintervall, wann die nächste Zufallsaktion kommt. Hier ist es anders: Jede Animation (Idle/Walk/Jump) hat eine eigene Dauer, und wenn sie zu Ende ist, löst sie selbst die nächste Zufallsauswahl aus — im Original, indem die letzte Frame der Zeitleisten-Animation MovieClip(this.parent).doNewAction() aufruft.

ActionScript 3 (Ende der Walk-Animation) 
stop();
trace(name + " walk done");
MovieClip(this.parent)
  .doNewAction();
JavaScript (im Update-Loop) 
if (this.t >= this.duration) {
  this.doNewAction();
}

Bewegung hängt vom Animationszustand ab

Wichtiges Detail aus CharakterControl.xml: Beim Laufen bewegt sich die Figur mit voller Geschwindigkeit, beim Springen nur mit der Hälfte, beim Stehen gar nicht. Bewegung und Animation sind also zwei getrennte, aber aufeinander abgestimmte Systeme — moveHorizontal() kümmert sich nur um Position und Randerkennung, welche Geschwindigkeit dabei gilt, entscheidet der aktuelle Animationszustand.

Bei Demo 2 laufen fünf Charaktere unabhängig voneinander — jeder mit eigenem Rhythmus zwischen Laufen, Stehen und Springen, jeder prallt eigenständig an den Bildschirmrändern ab.
