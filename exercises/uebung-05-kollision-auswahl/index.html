Kollision & Charakter-Auswahl

Erst die Grundlage jeder Kollisionserkennung, dann der große Schritt: Charaktere per Klick auswählen und gezielt mit der Tastatur steuern — über ein Event-Registrierungsmuster statt direkter Referenzen. Original: vl6.fla, uebung5_neu.fla.

hitTestObject(): Rechteck- oder Kreis-Überlappung

vl6.fla ist denkbar einfach: Ein Kreis folgt der Maus, ein zweiter steht fest, hitTestObject() prüft die Überlappung. Da Flashs hitTestObject() intern auf den Bounding-Boxen arbeitet, für zwei Kreise aber ein Abstandstest genauer ist, nutzt die Portierung direkt Math.hypot() gegen die Summe beider Radien — dasselbe Ergebnis, nur präziser für runde Formen.

Vom "alle machen etwas" zum "einer wird gesteuert"

Der eigentliche Sprung in diesem Kapitel: Bisher liefen alle Charaktere gleichzeitig mit eigenem Zufallsverhalten (Übung 3/4). Jetzt kann genau ein Charakter pro Zeitpunkt ausgewählt und mit der Tastatur gesteuert werden — alle anderen warten im Idle-Zustand.

Technisch spannend: Das Original löst die Auswahl nicht über eine direkte Referenz ("Hauptobjekt kennt Charakter X"), sondern über Events, die nach oben durchgereicht werden (dispatchEvent(new Event("registerCharacter", true)) mit bubbles = true). Jeder Charakter meldet sich beim Erzeugen selbst an, jeder Klick meldet sich selbst als "ausgewählt". Die Web-Entsprechung: ein gemeinsamer EventTarget als "Bus", auf dem alle Charaktere Events senden und ein zentraler Listener reagiert.

ActionScript 3 
function init() {
  var evt = new Event(
    "registerCharacter", true);
  dispatchEvent(evt);
}

function clicked(_event) {
  var evt = new Event(
    "characterClicked", true);
  dispatchEvent(evt);
}

// im Root:
stage.addEventListener(
  "registerCharacter",
  registerCharacter);
JavaScript 
// im Charakter-Konstruktor:
this.bus.dispatchEvent(
  new CustomEvent(
    "registerCharacter",
    { detail: this }));

// im zentralen Listener:
bus.addEventListener(
  "registerCharacter",
  e => characters.push(e.detail));

Dieses Muster hat einen echten Vorteil: Der Charakter muss nichts über seine Umgebung wissen — er meldet sich einfach an und kommuniziert über Events. Genau dieses Prinzip zieht sich durch den Rest des Kurses (Übung 6 und 8 fügen genauso "Level-Elemente" und "Münzen" per Event-Registrierung hinzu).

Nur der Sprung läuft ungestört durch

Wichtiges Detail aus dem Original: Während des Sprungs (state = "playing") unterbricht kein anderer Tastendruck die Animation, bis sie fertig ist (state = "stopped"). Das verhindert, dass ein Sprung durch schnelles Tastendrücken "abgeschnitten" wird — auch in dieser Portierung reagiert das Springen erst nach Ablauf seiner Dauer wieder auf neue Eingaben.

Bei Demo 2 einen der drei Charaktere anklicken (er bekommt einen gestrichelten Auswahlring), dann mit ←/→/Space steuern. Ein Klick auf einen anderen Charakter wechselt die Auswahl.
