Schwerkraft & Level-Kollision

Drei Bausteine: Schwerkraft mit Abprallen, Raster-Interaktion, und die Charakter-Auswahl aus Übung 5 mit echter Plattform-Kollision. Original: vl7.fla, vl7_2.fla, uebung6.fla.

Schwerkraft: eine Zeile, die alles verändert

vl7.fla zeigt Schwerkraft in ihrer einfachsten Form: deltaY += 1 pro Frame, addiert auf die Y-Position. Trifft das Objekt eine Plattform, wird deltaY auf einen negativen Wert zurückgesetzt — es prallt ab.

ActionScript 3 
function update(_event: Event) {
  this.y += deltaY;
  deltaY += 1;

  for(var i=0; i<level.numChildren; i++) {
    var hit = this.hitTestObject(
      level.getChildAt(i));
    if(hit) {
      deltaY = -10; // bouncing
    }
  }
}
JavaScript 
rect.deltaY += 1;
rect.y += rect.deltaY;

level.forEach(l => {
  const hit = /* AABB-Test */;
  if (hit && rect.deltaY > 0) {
    rect.deltaY = -10;
    rect.y = l.y - rect.h;
  }
});

Raster-Interaktion: Klick entfernt ein Element

vl7_2.fla erzeugt ein 10×10-Raster aus Rechtecken und entfernt bei Klick genau das getroffene Element — ein einfaches, aber wichtiges Muster für alles von Gitterspielen bis zu Inventar-Systemen.

Von der Rechteck-Kollision zur Level-Registrierung

uebung6.fla baut auf Übung 5 auf und fügt echte Boden-/Plattform-Kollision hinzu — über dieselbe Event-Registrierungskette wie schon bei den Charakteren: Collider/LevelBox-Objekte melden sich per RegisterCollider-Event an, LevelStage reicht das als RegisterLevelElement weiter nach oben. Der Charakter-Controller sammelt alle registrierten Elemente und prüft bei jedem Frame, ob sein "Fußpunkt" eines davon berührt.

ActionScript 3 
// LevelBox meldet sich an:
this.dispatchEvent(
  new Event("RegisterCollider", true));

// LevelStage reicht weiter:
function registerCollider(_event) {
  _event.target.dispatchEvent(
    new Event(
      "RegisterLevelElement", true));
}
JavaScript 
// LevelPlatform meldet sich an:
bus.dispatchEvent(
  new CustomEvent(
    "RegisterLevelElement",
    { detail: this }));

// zentraler Listener sammelt:
bus.addEventListener(
  "RegisterLevelElement",
  e => level.push(e.detail));

Der Vorteil bleibt derselbe wie in Übung 5: Ein Level kann beliebig viele Plattformen haben, ohne dass der Charakter-Code sie einzeln kennen muss — er fragt einfach "welche registrierten Elemente berühre ich gerade".

Bei Demo 3 einen Charakter anklicken, dann mit ←/→ über die Plattformen laufen und mit Space springen — nur möglich, während er auf dem Boden steht.
