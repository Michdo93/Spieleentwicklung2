OOP-Grundlagen

Klassen, Konstruktoren, Vererbung und Pakete — die Grundlage für saubere Spielearchitektur, ganz ohne Spiellogik erklärt. Original: Main.as, Test.as, Circle.as.

AS3-Klassen und ES6-Klassen: fast wortgleich

Das ist die gute Nachricht dieses Kapitels: AS3s Klassensyntax war stark an ECMAScript angelehnt — moderne JavaScript-Klassen (ES6) sehen fast identisch aus. Konstruktor, Vererbung, super() — alles hat eine direkte Entsprechung.

ActionScript 3 
package {
  import flash.display.MovieClip;

  public class Test extends MovieClip {
    public function Test() {
      trace("I am test");
      this.addEventListener(
        Event.ENTER_FRAME, this.update);
    }

    public function update(_event: Event) {
      this.x += 2;
    }
  }
}
JavaScript (ES6) 
class Test {
  constructor(x, y) {
    this.x = x;
    this.y = y;
    console.log("I am test");
  }

  update(dt) {
    this.x += 90 * dt;
  }
}

Vererbung: extends heißt in beiden Sprachen extends

Circle extends Test übernimmt automatisch update() — die Bewegung muss nicht neu geschrieben werden. Nur die Darstellung wird überschrieben (Kreis statt Quadrat). Das ist der Kernvorteil von Vererbung: gemeinsames Verhalten einmal definieren, spezialisiertes Verhalten in der Unterklasse ergänzen.

ActionScript 3 
public class Circle extends Test {
  public function Circle() {
    trace("I am circle");
  }
}
JavaScript 
class Circle extends Test {
  constructor(x, y, color) {
    super(x, y, color);
    console.log("I am circle");
  }

  draw(ctx) { /* Kreis statt Quadrat */ }
}

Beachte die Reihenfolge im Ausgabe-Panel rechts: zuerst "I am root" (Main wird erzeugt), dann für jede Circle-Instanz erst "I am test" (der Basisklassen-Konstruktor läuft zuerst) und danach "I am circle" — super() muss in JavaScript wie in AS3 aufgerufen werden, bevor der Rest des Konstruktors ausgeführt wird.

Pakete: package {} vs. ES-Module

AS3 organisierte Klassen in package-Namespaces (package packages.main.hello { ... }), aufgelöst über den Ordnerpfad der Datei. Die moderne JavaScript-Entsprechung sind ES-Module (export class Circle { ... } in einer eigenen Datei, import { Circle } from './Circle.js' anderswo) — dasselbe Ziel (Namensräume, Wiederverwendbarkeit über Dateien hinweg), nur ohne den Umweg über eine Paket-Deklaration. Für dieses kleine Demo reicht eine gemeinsame Datei, aber ab Übung 8 (dem großen Plattformer-Projekt) wird diese Aufteilung wichtig.

Rechts unten läuft ein Quadrat (Test) und ein Kreis (Circle) über die Bühne — beide bewegen sich exakt gleich, weil Circle die Bewegung von Test erbt. "Neu erzeugen" im Ausgabe-Panel zeigt die Konstruktor-Reihenfolge erneut.
