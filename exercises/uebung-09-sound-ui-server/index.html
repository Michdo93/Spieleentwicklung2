Sound, UI & Server

Drei für sich stehende Themen, die zu jeder echten Anwendung dazugehören: fertige UI-Bausteine, Soundeffekte, und Kommunikation mit einem Server. Original: vl10.fla, vl10_2.fla, vl11.fla.

Flash-UI-Komponenten → native HTML-Formularelemente

Animate brachte fertige UI-Bausteine mit (Stepper, Button, CheckBox, ColorPicker, Slider) — der Browser hat für praktisch jeden davon eine eingebaute Entsprechung, meist sogar barrierefreier und ohne zusätzlichen Code:

ActionScript 3 
stepper_mc.addEventListener(
  Event.CHANGE, getStep);
colorPicker_mc.addEventListener(
  ColorPickerEvent.CHANGE, getColor);
slider_mc.addEventListener(
  SliderEvent.CHANGE, getSlide);
HTML 
<input type="number">
<input type="color">
<input type="range">
// jeweils mit 'change'/'input'-Event

Sound: eingebettet vs. gestreamt

Das Original unterscheidet zwei Sound-Arten: ein in die Bibliothek eingebetteter Sound-Effekt (footstep, als eigene Klasse mit extends Sound exportiert) und ein von extern gestreamter Loop (SlowLoop.mp3) mit SoundTransform für die Lautstärke. Im Web verschwimmt dieser Unterschied — beides ist einfach eine Audiodatei, geladen über new Audio().

ActionScript 3 
var sound = new footstep();
sound.play();

var sndLoop = new Sound(
  new URLRequest("Sounds/SlowLoop.mp3"));
var channel = sndLoop.play(0, 10);
channel.soundTransform =
  new SoundTransform(0.8, 0);
JavaScript 
const footstep = new Audio(
  "assets/footstep.mp3");
footstep.play();

const loop = new Audio(
  "assets/SlowLoop.mp3");
loop.loop = true;
loop.volume = 0.8;
loop.play();

Beide Audiodateien sind hier Original-Assets — aus den FLA-Dateien extrahiert (sie lagen dort als eingebettete WAV vor) und nach MP3 konvertiert.

Server-Kommunikation: URLLoader → fetch()

Das Original schickt Daten an einen eigenen PHP-Endpunkt auf dem Hochschul-Webspace. Für eine portable, öffentlich lauffähige Demo wurde hier ein öffentlicher Test-Server (httpbin.org) eingesetzt, der Anfragen einfach zurück-echot — das Prinzip bleibt exakt gleich.

ActionScript 3 
var variables = new URLVariables();
variables["name"] = "Player1";

var request = new URLRequest(
  "https://.../server_communication.php");
request.data = variables;

var loader = new URLLoader();
loader.addEventListener(
  Event.COMPLETE, httpRequestComplete);
loader.load(request);
JavaScript 
const response = await fetch(
  "https://httpbin.org/post", {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ name })
  });
const data = await response.json();
Demo 3 braucht eine Internetverbindung — der Test-Server ist öffentlich und kann gelegentlich langsam antworten oder ausfallen, das Prinzip (Daten senden, Antwort verarbeiten) bleibt aber sichtbar.
